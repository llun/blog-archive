Spring MVC กับ Strut

By llun on November 19, 2006 in ภาษา

ที่จริงจะมาบ่นเรื่อง Spring MVC อ่ะ ก็เลยเอา Strut มาเปรียบเทียบด้วยเลย เพราะด้วยความยากของ Spring MVC ที่มันมีให้บ่นเยอะเหลือเกิน หรือเพราะชินกับการลากเส้นของ Strut ก็ไม่รู้ แต่ใช้ แล้วมันมีข้อข้องใจให้ต้องเปิดเว็บหาวิธีแก้เรื่อยไปอยู่ทุกทีสิ

อย่างแรกเลยที่เซ็งมากเกี่ยวกับ Spring MVC คือการ Forward เพราะถ้าเป็น Strut มันจะแค่บอก Forward แล้วก็ Action หรือ Page เลยตอน Return (โดยการลากเส้น) แต่ถ้าเป็น Spring ดันต้องสร้าง Redirect View แล้วบอกมันก่อนว่าจะให้ไป Controller ไหน ถ้าจะให้ชี้ไปที่ Action อีก Action นึง โอ้วพระเจ้าจอร์จ ทำไมไม่ทำให้มันเหมือนกันฟะ ระหว่างหน้า page กับ Controller -*- ยังไงมันก็ Forward ไปเหมือนกัน หรือเพราะมันไม่มี Tools แบบ Strut ฟะ เลยยาก

อย่างที่สอง Controller ของ Spring MVC มันมีซะโคตรหลายแบบเลย แถมเวลาใช้ก็ใช้ต่างกันด้วย มันชวนให้สับสนดีจริงๆ ถ้าเป็น Strut นะ อะไรๆ ใส่เข้า Action จบ -*-

อย่างที่ 3 คือเรื่อง Tag ของ Spring นั่นแล ไมมันมีให้น้อยจังฟะ แถมใช้ไม่ค่อยสะดวกเลย bind ตัวแปลได้ทีละตัว ทีละก้อนในหนึ่งบล๊อก พอจะใช้ก็ bind ใหม่ ไม่งั้นก็ต้องสร้างคลาสใหม่เพื่อจัดการกับ form นั้นโดยเฉพาะ (เอ strut ก็มีนี่หว่า form bean เย้ย ไม่ไช่ รู้แต่ว่าอันนี้มันลำบากกว่าตอน strut ยังไงไม่รุ)อาจจะเพราะไม่ชินหรือว่าไรดี แต่มันใช้แล้วลำบากไงไม่รู้ งานต่อไปลองมองๆ jsf หน่อยดีกว่า ไม่รู้จะลำบากแบบนี้ป่าวหว่า แต่มันมี Tool แบบลากแล้ววางหนินะ ไม่ต้องมาจินตนาการ flow เอง น่าจะง่ายกว่า อีกตัวก็คงเป็น spring work flow แต่ไว้ก่อน งานเสร็จแล้วค่อยลอง เห้อ
