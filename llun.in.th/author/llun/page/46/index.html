เขียนบล๊อกกันเถอะ

By llun on October 18, 2008 in มั่วๆ

หลังจากเขียนไปนิดหน่อยในเรื่องที่แล้วเลยรู้สึกอยากเขียนต่อ สิ่งที่มักเจอประจำเวลาทำงานคือมักเจอคำถามที่บางทีมันน่าจะรู้อยู่แล้วจากคนที่ทำงานอยู่ตรงนั้นเช่นเพื่อนเขียนโปรแกรมเกี่ยวกับฐานข้อมูล แต่เวลามีปัญหานิดหน่อยก็มักจะถามผม พอถามเสร็จได้คำตอบ(ค้นจาก Google ให้จริงๆ อันนี้เป็นอีกปัญหานึงที่อยากบ่น) ก็ไปทำต่อแก้เสร็จ ผ่านไปซักระยะ มาถามใหม่ – -” ปัญหาเดิมเอาง่ายๆ เลยจะดึงข้อมูลจากฐานข้อมูลขึ้นมายังไง คือถ้าทุกคนพอได้คำตอบแล้วหาที่เขียนซักที่ เวลาเจอปัญหาอีกครั้งแล้วค้นหาในที่ๆ ตัวเองจดไว้ เชื่อว่าปัญหาพวกนี้มันน่าจะลดลงได้ และที่ผมเห็นใกล้ตัวสุดและที่ใช้มารู้สึกได้ผลสุดคือบล๊อกนี่แหละ จริงๆ ที่ผมมีปัญหาที่สุดคือที่ที่ทำงานอยู่ตอนนี้นี่แหละ เพราะถึงแม้จะมีวิกิของโครงการต่างๆ ไว้ให้จดอยู่แล้ว แต่เวลาจะเข้าไปเขียนทีมันให้ความรู้สึกว่ามันไม่ใช่อ่ะ จะบอกว่ามันเขียนยากก็ไม่ใช่ แต่ในวิกิมันเขียนแล้วจะรู้สึกว่าเป็นวิชาการมากกว่า คือเขียนไม่ง่ายเหมือนบล๊อกว่่าอย่างนั้นเถอะ พอมันเขียนยาก ความอยากเขียนหรือจดบันทึกอะไรเกี่ยวกับสิ่งที่ทำ มันก็จะลดลงไป จนสุดท้ายก็เป็นว่าไม่ได้เขียนเลย แน่นอนบางเรื่องมันก็ควรรวบรวมมาเขียนเป็นวิกิ แต่ที่อยากได้คือเรื่องที่ง่ายๆ อย่างเช่นเวลาทำอะไรใหม่ๆ แล้วเจอปัญหาทีหรือวิธีแก้ที มันไม่น่าควรจะเอาไปเขียนในวิกิหรอกจริงมั้ย เช่น เวลาศึกษาอะไรใหม่กันที อย่าง Flex ทุกคนเริ่มจากศูนย์กันหมด แต่พอทำๆ กันไปแล้วทุกคนรู้มากขึ้น แต่ถ้าไม่เขียนถึงแม้มันจะมีหนังสือ เวลาเจอปัญหาทีก็ต้องไปค้นจาก Google ทีว่าไอ้นี่มันทำยังไง แม้มันจะคุ้นๆ ก็ตาม ตอนนี้คิดได้แค่นี้ก่อน แต่เหมือนมันยังมีอะไรค้างอยู่ในหัวอีก อย่างเช่น ถ้าให้แต่ละคนในบริษัทเขียนถึงงานที่ทำ และเล่าถึงปัญหามันควรจะเป็นบล๊อกภายในหรือป่าว แต่ถ้าทำอย่างนั้น [...]

Automatic ssh authenticate

By llun on October 18, 2008 in ระบบ

วันนี้ willwill ถามในห้อง #ubuntu-th ว่าทำยังไงให้เข้าไปในเซิร์ฟเวอร์ได้โดยไม่ต้องพิมพ์รหัสผ่าน ก็คุ้นๆว่าเคยทำเมื่อปีที่แล้ว แต่ลืมไปแล้วค้นๆ ในบล๊อกก็เคยทำจริงด้วยแต่ไม่ได้เขียนวิธีไว้วันนี้เลยขุดค้นใหม่ (แต่ willwill เจอวิธีไปก่อนและ แต่คนละเว็บ) เอามาเขียนเก็บกันลืมในนี้อีกที ขั้นแรกล๊อกอินเข้าไปที่เซิร์ฟเวอร์ผ่าน ssh แล้วสร้าง key มาก่อนด้วยคำสั่งssh-keygen -t rsaโดยด้านหลังสุดอาจเป็น rsa หรือ dsa ก็ได้แล้วแต่วิธีที่จะใช้เข้ารหัส จากนั้นมันจะถามคำถามมามากมายรวมทั้งให้ใส่รหัสอีกทีรวมทั้งไฟล์ที่จะไว้เก็บ key นี้ด้วย สุดท้ายแล้วจะได้ไฟล์ออกมาสองไฟล์ คือไฟล์ที่เราใส่ชื่อเข้าไปกับ อีกอันที่มี .pub (คิดว่าเป็นไฟล์ที่เก็บ public key) ออกมา คัดลอกข้อมูลในไฟล์ .pub ลงไฟล์ ~/.ssh/authorized_keyscat host.pub >> ~/.ssh/authorized_keys เอาไฟล์ที่ได้ออกมาอีกไฟล์ส่งกลับมาที่เครื่องส่วนตัว แล้วเก็บไว้ที่ ~/.ssh เหมือนกัน ครั้งต่อไปที่เข้าเซิร์ฟเวอร์ก็ไม่ต้องใส่รหัสผ่านอีกต่อไปและ ช่วงนี้มานั่งนึกๆ ดูรู้สึกที่ทำงานมาหนึ่งปีกว่า ส่วนที่จำได้ส่วนใหญ่แล้วเกี่ยวกับงานมันก็จะถูกเขียนอยู่ในบล๊อกนี้แหละ ก็เลยอยากให้คนอื่นเขียนๆ ไว้บ้าง เอ๊ะ หรือว่าให้พี่สร้างบล๊อกของแต่ละคนในที่ทำงานไว้ดี เวลาทำไรก็ให้ทุกคนจดไว้ในนั้น [...]

อาทิตย์นี้ทั้งอาทิตย์วุ่นๆ ทั้งอาทิตย์เพราะจัดการแฟ้มๆ ต่างๆ ในเซิร์ฟเวอร์ใหม่หมดจากเดิมที่ข้อมูลโดเมนและเว็บแต่ละคนจะอยู่ในแฟ้มบ้านแต่ละคนตอนนี้ย้ายมาอยู่ที่เดียวกัน แล้วคนที่เข้ามาแก้ไขข้อมูลก็อยู่ในแฟ้มโดเมนนั้นไปเลย ไม่ให้ออกมาเพ่นพ่านแฟ้มคนอื่น ตอนแรกก็คิดว่ามันคงไม่ยากอะไร เพราะตั้งใจจะย้ายไปใช้ Lighttpd แต่ปรากฏว่ามันดันใช้ .htaccess เพื่อแก้ url แบบ Apache ไม่ได้พวกค่่าต่างๆ ที่ใช้ใน WordPress และโปรแกรมอื่นๆ พังหมด -_-” พยายามหาทางแก้ข้ามวันข้ามคืนว่า Lighttpd มีทางออกที่ดีกว่าไปเขียน url ใหม่ที่เป็นแบบ Lighttpd โดยเฉพาะหรือป่าว ก็ไม่มีสุดท้ายเลยยอมแพ้ กลับมาใช้ Apache เหมือนเดิม ที่อยากใช้ Lighttpd มากเพราะนอกจากมันเบาแล้วมันยังมีอุปกรณ์เสริมอีกตัวนึงที่อยากได้ใน Apache แต่ดันไม่มี(จริงๆ เหมือนจะมีแต่กำหนดได้งี่เง่ามาก) นั่นคือ mod_evhost เนื่องจากเซิร์ฟเวอร์ที่ดูแลอยู่มันยังไม่มีโปรแกรมที่ใช้ควบคุมเหมือนคนอื่น(พวก Direct Admin, Plesk, cPanel ไรพวกนั้นหนะ) ด้วยเหตุผลว่าจะทำเอง(จนพวกนั้นมันแพง -_-”) ก็เลยต้องหาทางให้แก้ไขไฟล์ต่างๆของเซิร์ฟเวอร์ให้น้อยที่สุด จะได้ไม่ต้องไปรีโหลด หรือเปิดปิดเว็บเซิร์ฟเวอร์บ่อยๆ แล้วโมดูลตัวนี้มันสามารถกำหนดโดยใช้พวก %1 … ต่างๆได้เพื่อกำหนดไปว่าถ้ามีคนเปิดเว็บโดเมนนี้ [...]
